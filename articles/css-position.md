# 🔹定位

- **作用**：灵活改变盒子在网页中的位置

- **实现条件**：定位模式 + 边偏移，二者结合才能生效
  1. 定位模式：通过`position`属性，设置定位模式
  2. 边偏移：设置盒子具体位置，可选`left`/`right`/`top`/`bottom`




## 📌定位模式



### ✨相对定位

- **语法：**`position: relative`

- **核心特点：**不脱标，占用自身原来的位置；显示模式保持标签原有特点

- **参照物**：相对于自己原来的位置进行移动

  


### ✨绝对定位

- **语法**：`position: absolute`

- **核心特点：**脱标，不占用页面位置；显示模式具备行内块特点
- **参照物：**优先相对最近的**已定位（非static）** 祖先元素，若无则相对浏览器可视区



### ✨固定定位

- **语法：**`position: fixed`

- **核心特点**：脱标，不占用页面位置；显示模式具备行内块特点
- **参照物：**始终相对**浏览器窗口**定位



## 📌堆叠层级 

- **语法：**`z-index`

- **作用**：设置**定位元素**的层级顺序，改变重叠时的显示优先级
- **默认规则**：定位元素重叠时，按标签书写顺序，后来者居上
- **属性值**：整数数字（默认值为0），取值越大，层级越高
- **注意**：仅对**已定位（position非static）** 的元素生效



# 🔹CSS 精灵

**CSS 精灵（CSS Sprites）**，是网页开发中专门处理**背景小图片**的核心优化技术，核心是将多张零散的背景小图整合到一张大图中，通过 CSS 背景定位精准显示单张小图。



## 📌核心定义与原理

- **定义：**把网页中需要用到的**多个背景小图标 / 小图片**整合到**一张单独的图片文件**中，页面加载时仅需请求这一张整合后的图片，再通过 CSS 的`background-position`属性，精准定位到目标小图的位置，让盒子只显示该区域的内容。

- **原理：**浏览器对每个网页资源的请求都有次数限制，加载多张零散小图会发起多次服务器请求，而 CSS 精灵将多图合一，**仅发起 1 次请求即可加载所有小图**，剩余的小图显示通过本地 CSS 定位实现，从根本上减少请求开销。
